## 清理工具

本次清理创建了以下工具脚本：

1. `cleanup_redundant_files.py` - 基础冗余文件清理
2. `deep_cleanup.py` - 深度清理脚本
3. `organize_docs.py` - 文档整理脚本
4. `organize_scripts.py` - 脚本整理脚本

这些工具可以在未来需要时重复使用。

## 维护建议

### 日常开发
1. 新的核心功能代码放在根目录
2. 新的文档按类型放入 `/docs` 对应子目录
3. 新的工具脚本放入 `/scripts` 对应子目录
4. 新的测试文件放入 `/tests` 对应子目录

### 定期清理
1. 每月运行一次 `deep_cleanup.py` 检查冗余文件
2. 及时删除临时测试文件和调试输出
3. 保持 `.gitignore` 更新，避免提交临时文件

### 文档管理
1. 任务完成后及时整理文档到 `/docs/tasks`
2. 分析报告放入 `/docs/analysis`
3. 重要的项目文档保持在根目录
